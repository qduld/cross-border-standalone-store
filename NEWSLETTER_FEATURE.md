# 订阅功能和群发邮件系统

## 🎉 功能概述

已实现完整的订阅和群发邮件系统，用户可以订阅你的邮件列表，你也可以向所有订阅者群发邮件。

## ✅ 已完成的功能

### 1. 用户订阅功能
- ✅ 订阅表单（首页底部）
- ✅ 邮箱验证
- ✅ 防止重复订阅
- ✅ 订阅成功/失败提示

### 2. 群发邮件功能
- ✅ 管理页面：`/admin/newsletter`
- ✅ 支持测试邮件（发送到指定邮箱测试）
- ✅ 支持群发邮件（发送到所有订阅者）
- ✅ 显示订阅者数量
- ✅ 精美的邮件模板（红色渐变主题）
- ✅ 分批发送（每批 50 封，避免被限制）

### 3. 数据库模型
- ✅ Subscriber 表（存储订阅邮箱）
- ✅ 支持激活/停用状态

## 📝 使用说明

### 用户订阅

1. 访问网站首页
2. 滚动到页面底部
3. 输入邮箱地址
4. 点击"订阅"按钮
5. 看到成功提示即完成订阅

### 管理员群发邮件

1. 访问：`/admin/newsletter`
2. 填写邮件主题
3. 填写邮件内容（支持 HTML）
4. **建议先发送测试邮件**：
   - 点击"先发送测试邮件"
   - 输入你的测试邮箱（如 `test@example.com`）
   - 点击"发送测试邮件"
   - 检查邮箱确认格式正确
5. 点击"群发邮件"按钮
6. 等待发送完成

## 🔧 技术细节

### API 端点

- `POST /api/subscribe` - 添加订阅
- `GET /api/subscribe` - 获取订阅者列表
- `DELETE /api/subscribe` - 取消订阅
- `POST /api/admin/newsletter` - 群发邮件

### 数据库

```prisma
model Subscriber {
  id        String   @id @default(cuid())
  email     String   @unique
  active    Boolean  @default(true)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

## ⚠️ 注意事项

### 关于 Prisma 错误

如果构建时看到 Prisma 相关错误，不用担心。Zeabur 上应该已经有正确的数据库配置（之前的 Stripe 订单功能已经在使用数据库）。

如果 Zeabur 部署后仍然有错误，请：
1. 检查 Zeabur 环境变量中的 `DATABASE_URL`
2. 确保数据库连接正常

### 邮件发送限制

飞书邮箱可能有以下限制：
- 每日发送数量限制
- 发送频率限制

解决方案：
- 分批发送（已实现，每批 50 封）
- 避免短时间内大量发送
- 如需大量发送，建议使用专业邮件服务（SendGrid、Resend）

### 邮件内容建议

✅ 推荐：
- 产品上新通知
- 限时优惠活动
- 节日促销
- 品牌故事分享

❌ 避免：
- 过于频繁（建议每月 1-2 次）
- 纯推销无价值内容
- 过长的邮件内容

## 🆘 遇到问题？

### 订阅失败
- 检查邮箱格式是否正确
- 检查是否已订阅
- 查看浏览器控制台错误

### 邮件未收到
- 检查垃圾邮件文件夹
- 检查飞书邮箱 SMTP 配置
- 查看 Zeabur 日志

### 群发失败
- 确认环境变量已配置
- 查看测试邮件是否正常发送
- 检查订阅者列表是否为空
